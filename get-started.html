Get Started with Claude Code via AWS Bedrock

Step-by-step guide to deploying FedRAMP High compliant AI development for Honeywell's DevSecOps teams

Implementation Overview

This guide walks you through setting up Claude Code access via AWS Bedrock in AWS GovCloud (US) to ensure FedRAMP High compliance. The entire process typically takes 1-2 weeks from start to production.

Days 1-2: Prerequisites & AWS Setup

Configure AWS GovCloud account and IAM permissions

Days 3-4: Bedrock Configuration

Enable Claude models and configure guardrails

Week 1-2: Pilot Deployment

Deploy to 10-20 engineers for testing

Week 3+: Scale & Optimize

Roll out to full team and integrate with existing tooling

Step-by-Step Implementation

1

Set Up AWS GovCloud Account

AWS GovCloud (US) is required for FedRAMP High authorization. If you don't already have a GovCloud account, you'll need to create one.

Prerequisites:

  • U.S. citizenship or entity requirement
  • Valid business use case for GovCloud
  • Existing AWS commercial account (recommended)

Steps:

  1. Navigate to AWS GovCloud homepage
  2. Click "Create an AWS GovCloud (US) account"
  3. Complete the registration form with required information
  4. Verify your identity and wait for account approval (typically 1-2 business days)
  5. Once approved, log in to your GovCloud account at: console.amazonaws-us-gov.com
üí° Tip: GovCloud accounts are separate from commercial AWS accounts. You'll receive different credentials and access the console at a different URL.
2

Configure IAM Permissions

Set up proper Identity and Access Management (IAM) permissions to allow engineers to access Amazon Bedrock and Claude models.

Create IAM Policy for Bedrock Access:

{ "Version": "2012-10-17", "Statement": [ { "Sid": "BedrockFullAccess", "Effect": "Allow", "Action": [ "bedrock:InvokeModel", "bedrock:InvokeModelWithResponseStream", "bedrock:ListFoundationModels", "bedrock:GetFoundationModel" ], "Resource": "*" }, { "Sid": "BedrockGuardrails", "Effect": "Allow", "Action": [ "bedrock:GetGuardrail", "bedrock:ListGuardrails" ], "Resource": "*" } ] }

Steps:

  1. Log in to AWS GovCloud IAM Console
  2. Navigate to IAM ‚Üí Policies ‚Üí Create Policy
  3. Select JSON tab and paste the policy above
  4. Name it: HoneywellClaudeBedrockAccess
  5. Create IAM role or attach policy to existing user groups
  6. Assign this policy to DevSecOps engineers who need Claude access
‚ö†Ô∏è Security Note: Use least-privilege access. Consider creating separate roles for different use cases (development vs. production) and implement MFA for all users.
3

Enable Amazon Bedrock in GovCloud

Amazon Bedrock must be enabled in your GovCloud account before you can access Claude models.

Steps:

  1. Log in to AWS GovCloud Console: console.amazonaws-us-gov.com
  2. Navigate to Amazon Bedrock service
  3. Select region: us-gov-west-1 (AWS GovCloud US-West)
  4. Click "Get Started"
  5. Review and accept the Amazon Bedrock service terms
üí° Region Note: Claude models in GovCloud are currently available in us-gov-west-1. Make sure to select this region for all Bedrock operations.
4

Request Model Access for Claude

Claude models require explicit access approval within Amazon Bedrock.

Steps:

  1. In Amazon Bedrock console, navigate to Model Access (left sidebar)
  2. Click "Manage Model Access"
  3. Find and select the following models:
    • ‚úì Claude 3.5 Sonnet v1 (anthropic.claude-3-5-sonnet-20240620-v1:0)
    • ‚úì Claude 3 Haiku (anthropic.claude-3-haiku-20240307-v1:0)
  4. Click "Request Model Access"
  5. Fill out the use case form describing your DevSecOps application
  6. Submit the request
üí° Approval Time: Model access is typically approved within minutes for GovCloud accounts. Check the "Model Access" page to confirm status shows "Access granted" before proceeding.
5

Configure Bedrock Guardrails (Recommended)

Implement guardrails to enforce security policies and content filtering for your team's AI usage.

Recommended Guardrail Policies:

  • Content Filters: Block inappropriate content, PII, and sensitive data
  • Denied Topics: Prevent discussions of unrelated topics
  • Word Filters: Block specific keywords or patterns
  • Sensitive Information: Redact SSNs, credit cards, API keys

Steps:

  1. In Bedrock console, navigate to Guardrails
  2. Click "Create Guardrail"
  3. Name: HoneywellDevSecOpsGuardrail
  4. Configure content filters:
    • Hate speech: MEDIUM
    • Violence: MEDIUM
    • Sexual content: HIGH
    • Misconduct: MEDIUM
  5. Add sensitive information filters:
    • ‚òëÔ∏è US Social Security Numbers
    • ‚òëÔ∏è Credit card numbers
    • ‚òëÔ∏è Email addresses (if needed)
    • ‚òëÔ∏è API keys and secrets
  6. Click "Create" and note the Guardrail ID
6

Test API Access

Verify that your configuration works by making a test API call to Claude via Bedrock.

Using AWS CLI:

aws bedrock-runtime invoke-model \ --region us-gov-west-1 \ --model-id anthropic.claude-3-5-sonnet-20240620-v1:0 \ --body '{ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 1024, "messages": [ { "role": "user", "content": "Write a secure Python function to validate user input." } ] }' \ --cli-binary-format raw-in-base64-out \ output.json cat output.json

Using Python (boto3):

import boto3 import json # Initialize Bedrock client for GovCloud bedrock = boto3.client( service_name='bedrock-runtime', region_name='us-gov-west-1' ) # Prepare request body = json.dumps({ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 1024, "messages": [ { "role": "user", "content": "Write a secure Python function to validate user input." } ] }) # Invoke Claude response = bedrock.invoke_model( modelId='anthropic.claude-3-5-sonnet-20240620-v1:0', body=body ) # Parse response response_body = json.loads(response['body'].read()) print(response_body['content'][0]['text'])
üí° Success Indicator: If you receive a response from Claude with generated code, your setup is working correctly!

8

Implement Monitoring & Cost Controls

Set up monitoring and cost controls to track usage and prevent unexpected charges.

CloudWatch Metrics to Monitor:

  • InvocationCount: Number of API calls to Claude
  • InputTokenCount: Tokens sent to the model
  • OutputTokenCount: Tokens generated by the model
  • InvocationLatency: Response time
  • InvocationErrors: Failed requests

Set Up Cost Alerts:

  1. Navigate to AWS Budgets in AWS Console
  2. Create a new budget: "Honeywell Bedrock Budget"
  3. Set monthly budget threshold (e.g., $10,000 for pilot)
  4. Configure alerts at 50%, 75%, and 90% of budget
  5. Add email notifications to DevSecOps leadership
‚ö†Ô∏è Cost Management: Claude 3.5 Sonnet v1 pricing in GovCloud:
  • Input: $3.00 per million tokens
  • Output: $15.00 per million tokens
Monitor usage closely during pilot phase. Implement caching and prompt optimization to reduce costs.
